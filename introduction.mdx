## Overview

Welcome to Roman Rudakov's documentation space. You get a centralized hub to organize, manage, and collaborate on all your project documentation. Build structured guides, API references, and changelogs effortlessly while keeping everything searchable and versioned.

This platform empowers you to create professional docs without complexity. Focus on content while it handles rendering, navigation, and updates.

## Get Started in Minutes

Follow these steps to set up your first documentation space.

<Steps>
  <Step title="Create Your Space" icon="plus">
    Sign up at `https://romanrudakov.com/docs` and create a new documentation workspace.
  </Step>
  <Step title="Add Your First Page" icon="file-plus">
    Click "New Page" and select `introduction.mdx`. Use the editor to add content.
  </Step>
  <Step title="Customize Navigation" icon="settings">
    Edit the sidebar in workspace settings to organize pages like `/quickstart` and `/api`.
  </Step>
  <Step title="Publish and Share" icon="share-2">
    Hit publish. Share the live URL with your team.
  </Step>
</Steps>

## Navigation and Structure Guide

Explore how to navigate and structure your docs effectively.

<Tabs>
  <Tab title="Side Navigation" icon="sidebar">
    Use the left sidebar for quick jumps. Organize into sections like "Guides" and "References".

    Customize via YAML frontmatter and sidebar config files.
  </Tab>
  <Tab title="Search and Breadcrumbs" icon="navigation-2">
    Top search bar finds pages instantly. Breadcrumbs show your location: Home > Guides > Quickstart.
  </Tab>
</Tabs>

<Expandable title="Advanced Structure Tips" default-open="false">

Structure pages hierarchically:

- Use H2 (`##`) for main sections
- Link internally with `/quickstart`
- Embed components for rich content

| Element       | Purpose                  |
|---------------|--------------------------|
| Frontmatter   | Sets title and metadata |
| H2 Headings   | Main sections            |
| Components    | Interactive elements     |

</Expandable>
