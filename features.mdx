## Document Creation and Editing

Start new documents directly from the dashboard. You edit using a live preview markdown editor that supports MDX components for interactive content.

<Steps>
  <Step title="Create a New Page" icon="plus">
    Click the `New Page` button. Choose a template or start blank.
  </Step>
  <Step title="Edit Content">
    Use markdown syntax. Add components like `<Callout>` for notes.

````markdown
# My Feature Page

<Callout kind="info">
  This is an important note.
</Callout>

```javascript
console.log("Hello, Roman Rudakov Documentation!");
```
````

  </Step>
  <Step title="Publish">
    Save and publish. Changes go live instantly.
  </Step>
</Steps>

<Callout kind="tip">
  Enable version history to track edits and revert changes easily.
</Callout>

## Organization with Folders and Tags

Keep your docs structured. Folders create hierarchies, while tags enable cross-referencing.

<Tabs>
  <Tab title="Folders" icon="folder">
    Nest pages like a file system.

    ```
    /project-docs/
      /api/
        endpoints.mdx
      /guides/
        quickstart.mdx
    ```
  </Tab>
  <Tab title="Tags" icon="tag">
    Add tags like `api`, `guide` for filtering.

    <CodeGroup tabs="Sidebar,Search">
    ````markdown
    ---
    tags: ["api", "feature"]
    ---
    ````
    ````
    ````bash
    Search: api feature
    ````
    </CodeGroup>
  </Tab>
</Tabs>

## Search Functionality

Powerful search finds content instantly. Use keywords, tags, or advanced filters.

| Feature | Description |
|---------|-------------|
| Full-text | Matches across titles, content, code. |
| Tag filter | Narrow by `api` or `guide`. |
| Folders | Scope to specific areas. |
