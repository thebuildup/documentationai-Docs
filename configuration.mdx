## Branding with Custom Colors

Apply your brand identity by setting custom colors. Use the primary color `#3B82F6` as shown in your product information.

<Steps>
  <Step title="Access Branding Panel" icon="palette">
    Navigate to **Space Settings > Branding** in the dashboard.
  </Step>
  <Step title="Update Primary Color">
    Enter your hex code, such as `#3B82F6`, in the Primary Color field.
  </Step>
  <Step title="Preview and Save">
    Preview changes live, then click **Save Branding**.
  </Step>
</Steps>

For programmatic updates, configure a `config.json` file:

<CodeGroup tabs="JSON,YAML">
  ```json
  {
    "branding": {
      "primaryColor": "#3B82F6",
      "secondaryColor": "#1E40AF"
    }
  }
  ```
  ```yaml
  branding:
    primaryColor: "#3B82F6"
    secondaryColor: "#1E40AF"
  ```
</CodeGroup>

<Callout kind="tip">
  Test colors in different themes to ensure readability. Use tools like contrast checkers for accessibility.
</Callout>

<ParamField path="branding.primaryColor" param-type="string" required="true">
  Hex color code for your primary brand color, e.g., `#3B82F6`.
</ParamField>

## User Roles and Permissions

Define roles like Admin, Editor, and Viewer to control access.

<Tabs>
  <Tab title="Admin" icon="shield">
    Full access: Edit content, manage users, configure spaces.
  </Tab>
  <Tab title="Editor" icon="edit-3">
    Edit documents and settings, but cannot delete spaces.
  </Tab>
  <Tab title="Viewer" icon="eye">
    Read-only access to documentation.
  </Tab>
</Tabs>

Assign roles via the API:

```javascript
await fetch('https://api.example.com/v1/spaces/{spaceId}/users', {
  method: 'POST',
  headers: { 'Authorization': 'Bearer YOUR_TOKEN' },
  body: JSON.stringify({
    userId: 'user-123',
    role: 'editor'
  })
});
```

## Backup and Export Configurations

Schedule automatic backups and export formats.

| Option       | Description                          | Frequency Options     |
|--------------|--------------------------------------|-----------------------|
| Daily Backup | Full space snapshot to S3            | Daily, Weekly         |
| Export      | PDF, Markdown, HTML bundles          | Manual, Scheduled     |

<Callout kind="alert">
  Enable backups before major changes. Exports include all versions.
</Callout>

Use the API for exports:

<Request tabs="cURL,JavaScript">
  ```bash
  curl -X POST https://api.example.com/v1/spaces/{spaceId}/export \
    -H "Authorization: Bearer YOUR_TOKEN" \
    -d '{"format": "markdown"}'
  ```
  ```javascript
  const response = await fetch(`https://api.example.com/v1/spaces/${spaceId}/export`, {
    method: 'POST',
    headers: { 'Authorization': `Bearer ${YOUR_TOKEN}` },
    body: JSON.stringify({ format: 'markdown' })
  });
  ```
</Request>

<Response tabs="200">
  ```json
  {
    "status": "success",
    "downloadUrl": "https://api.example.com/downloads/export-123.zip"
  }
  ```
</Response>

<ParamField header="Authorization" param-type="string" required="true">
  Bearer token for authentication.
</ParamField>
